## 1.1 C++ 是什么？和 Lua 有什么区别？

**你写传奇脚本用的是 Lua，C++ 是另一个层面的工具。**

| 对比 | Lua | C++ |
|------|-----|-----|
| 运行方式 | 脚本语言，边解释边执行 | 编译型语言，先翻译成机器码再执行 |
| 速度 | 较慢（需要虚拟机） | 极快（直接跑机器码） |
| 内存管理 | 自动垃圾回收（GC） | 手动管理（但更精准） |
| 类型检查 | 动态类型 | 静态类型（先声明再使用） |
| 你用 Lua 做什么 | 游戏逻辑、协议处理 | 游戏逻辑层的底层支撑 |

**C++ 的优势：**
- 游戏引擎核心（Unity、Unreal）都是 C++
- 游戏服务器用 C++ 性能最好
- 学了 C++，你 Lua 脚本的底层原理就清楚了

## 1.2 第一个程序：编译和运行

**C++ 程序从写好到运行，分两步：**

```
第1步：编译（翻译成机器码）
  hello.cpp  →  hello.exe

第2步：运行
  hello.exe
```

**你电脑上的编译方法：**

在 PowerShell 中运行：
```powershell
powershell -ExecutionPolicy Bypass -File C:\cpp\compile.ps1 C:\cpp\hello.cpp
```

**编译脚本的原理（了解一下即可）：**
```powershell
# 实际上执行的是这一行：
cl /EHsc /std:c++17 hello.cpp /Fe:hello.exe
# cl      = MSVC 编译器
# /EHsc   = 启用异常处理
# /std    = 使用 C++17 标准
# /Fe     = 指定输出文件名
```

**C++ 程序的基本结构：**
```cpp
#include <iostream>      // 引入输入输出工具（相当于 Lua 的 require）
using namespace std;   // 使用标准工具箱

int main() {            // 程序入口，所有代码从这里开始
    cout << "Hello" << endl;
    return 0;          // 返回 0 表示正常结束
}
```

**和 Lua 对比：**
```lua
-- Lua：一行代码搞定
print("Hello")

-- C++：需要更多仪式感，但编译器能帮你检查错误
```

## 1.3 变量：给数据起名字

**Lua 的变量：**
```lua
name = "战士"
level = 30
hp = 100.0
-- Lua 自动识别类型，随时可以改变
name = 100  -- 合法，类型可以随时变
```

**C++ 的变量（必须先声明类型）：**
```cpp
string name = "战士";   // 字符串
int level = 30;          // 整数
double hp = 100.0;       // 浮点数
```

**为什么 C++ 要声明类型？**
1. 告诉计算机分配多少内存
2. 编译时就检查错误，而不是运行到一半崩溃
3. 编译器能优化，运行更快

**C++ 类型一旦声明，不能随意改变：**
```cpp
int x = 10;
x = 20;       // 合法，20 还是整数
x = "hello";  // 错误！编译时就报错
```

## 1.4 C++ 常用数据类型

| 类型 | 含义 | 举例 | 占用 |
|------|------|------|------|
| `int` | 整数 | `30`, `-10`, `0` | 4字节 |
| `double` | 浮点数（小数） | `100.5`, `3.14` | 8字节 |
| `float` | 单精度浮点 | `172.5f` | 4字节 |
| `char` | 单个字符 | `'A'`, `'好'` | 1字节 |
| `bool` | 布尔（真假） | `true`, `false` | 1字节 |
| `string` | 字符串 | `"战士"` | 动态 |

**和 Lua 类型的对应关系：**
```
Lua 类型           C++ 类型
────────────────────────────
nil                 无对应
number（整数）      int
number（小数）       double
string              string（需 #include <string>）
boolean             bool
table               多种组合（数组+结构体+map）
```

## 【实战 1-2B】用变量描述一个玩家

**任务：** 用 C++ 变量描述一个玩家的基本信息并打印出来

**要求：**
- 玩家名字（字符串）
- 玩家等级（整数）
- 玩家生命值（浮点数）
- 是否在线（布尔）

```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    // 定义玩家数据
    string playerName = "战士";
    int playerLevel = 30;
    double playerHp = 100.0;
    bool isOnline = true;

    // 打印玩家信息
    cout << "===== 玩家信息 =====" << endl;
    cout << "名字: " << playerName << endl;
    cout << "等级: " << playerLevel << endl;
    cout << "HP: " << playerHp << endl;
    cout << "在线: " << (isOnline ? "是" : "否") << endl;

    return 0;
}
```

**输出：**
```
===== 玩家信息 =====
名字: 战士
等级: 30
HP: 100
在线: 是
```

**注意：** `isOnline ? "是" : "否"` 是三元运算符，等价于：
```cpp
if (isOnline) cout << "是";
else cout << "否";
```

## 1.5 cout 输出和 cin 输入

**cout（打印到屏幕）：**
```cpp
cout << "你好" << endl;           // 打印字符串，换行
cout << 123 << endl;             // 打印数字
cout << "等级:" << level << endl; // 拼接打印
```

**cin（从键盘读取）：**
```cpp
int age;
cout << "输入年龄:";
cin >> age;          // 读取整数到 age
cout << "你的年龄是 " << age << endl;

string name;
cout << "输入名字:";
cin >> name;         // 读取字符串到 name
```

**cin 的注意事项：**
- `cin >>` 会跳过空格和换行
- 如果要读一整行（包括空格），用 `getline(cin, str)`

## 【实战 1-2C】创建玩家（用户输入）

**任务：** 让用户输入玩家信息，创建角色


## 1.6 变量命名规则和习惯

**C++ 命名规则：**
- 字母、数字、下划线组成
- 不能以数字开头
- 不能用保留字（如 `int`、`for`、`while`）

**好习惯（和 Lua 的区别）：**
```cpp
// C++ 常用小写下划线（Linux 风格）
string player_name = "战士";
int player_level = 30;

// C++ 也常用驼峰命名（Java 风格）
string playerName = "战士";
int playerLevel = 30;

// Lua 习惯用下划线
player_name = "战士"
player_level = 30
```

**本教程统一用小写下划线风格**（`player_name`、`player_level`），这是 C++ 社区的主流风格之一。

## 【实战 1-2D】玩家升级系统

**任务：** 输入玩家当前等级，打印升级后的等级

```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    int level;

    cout << "===== 玩家升级系统 =====" << endl;
    cout << "当前等级: ";
    cin >> level;

    // 升级：等级 + 1
    level = level + 1;

    cout << "升级后等级: " << level << endl;

    // 简短写法
    level++;  // 等价于 level = level + 1
    cout << "再升一级: " << level << endl;

    return 0;
}
```
