# 第1章：变量与数据类型

## 1.1 C++ 是什么？和 Lua 有什么区别？

**你写传奇脚本用的是 Lua，C++ 是另一个层面的工具。**

| 对比 | Lua | C++ |
|------|-----|-----|
| 运行方式 | 脚本语言，边解释边执行 | 编译型语言，先翻译成机器码再执行 |
| 速度 | 较慢（需要虚拟机） | 极快（直接跑机器码） |
| 内存管理 | 自动GC | 手动管理（但更精准） |
| 类型检查 | 动态类型 | 静态类型（先声明再使用） |

**学 C++ 的理由：**
- 游戏引擎核心（Unity、Unreal）都是 C++
- 游戏服务器用 C++ 性能最好
- 懂了 C++，你 Lua 脚本的底层原理就清楚了

## 1.2 第一个程序：编译和运行

**C++ 程序分两步：**

```
第1步：编译
  hello.cpp  →  hello.exe

第2步：运行
  hello.exe
```

**在你的电脑上编译运行：**

```powershell
powershell -ExecutionPolicy Bypass -File C:\cpp\compile.ps1 C:\cpp\hello.cpp
```

看到 `Hello, World!` 输出即成功。

## 1.3 HelloWorld 逐行解析

```cpp
#include <iostream>      // 引入输入输出工具（相当于 Lua 的 require）
using namespace std;   // 使用标准命名空间

int main() {            // 程序入口，从这里开始执行
    cout << "Hello, World!" << endl;
    return 0;           // 返回 0 表示正常结束
}
```

**逐行解释：**

- `#include <iostream>`：把 iostream 文件内容复制进来，提供 `cout`、`cin`、`endl`
- `using namespace std;`：让 `cout` 不用写成 `std::cout`
- `int main() { }`：主函数，程序入口
- `cout << "Hello" << endl;`：打印内容到屏幕，`endl` = 换行

**和 Lua 对比：**
```lua
-- Lua：一行代码搞定
print("Hello")

-- C++：需要更多仪式感，但编译器能帮你提前检查错误
```

## 1.4 变量：给数据起名字

**Lua 的变量：**
```lua
name = "战士"
level = 30
hp = 100.0
-- Lua 自动识别类型，随时可以改变
name = 100  -- 合法，类型可以随时变
```

**C++ 的变量（必须先声明类型）：**
```cpp
string name = "战士";   // 字符串
int level = 30;          // 整数
double hp = 100.0;       // 浮点数
```

**为什么 C++ 要声明类型？**
1. 告诉计算机分配多少内存
2. 编译时就检查错误，而不是运行到一半崩溃
3. 编译器能优化，运行更快

**类型一旦声明，不能随意改变：**
```cpp
int x = 10;
x = 20;       // 合法，20 还是整数
x = "hello";  // 错误！编译时就报错
```

## 1.5 C++ 常用数据类型

| 类型 | 含义 | 举例 | 占用 |
|------|------|------|------|
| `int` | 整数 | `30`, `-10`, `0` | 4字节 |
| `double` | 浮点数（小数） | `100.5`, `3.14` | 8字节 |
| `float` | 单精度浮点 | `172.5f` | 4字节 |
| `char` | 单个字符 | `'A'`, `'好'` | 1字节 |
| `bool` | 布尔（真假） | `true`, `false` | 1字节 |
| `string` | 字符串 | `"战士"` | 动态 |

**和 Lua 类型的对应关系：**

```
Lua 类型           C++ 类型
────────────────────────────
nil                 无对应
number（整数）      int
number（小数）       double
string              string（需 #include <string>）
boolean             bool
table               多种组合（数组+结构体+map）
```

## 1.6 cout 输出

```cpp
cout << "你好" << endl;           // 打印字符串，换行
cout << 123 << endl;             // 打印数字
cout << "等级:" << level << endl; // 拼接打印，多个 << 连用
```

**`<<` 是插入运算符，把内容"流"向屏幕。**

---

## 1.7 cin 输入

```cpp
int age;
cout << "输入年龄:";
cin >> age;          // 读取整数到 age
cout << "你的年龄是 " << age << endl;

string name;
cout << "输入名字:";
cin >> name;         // 读取字符串到 name
```

**`>>` 是提取运算符，从键盘"流"入变量。**

**注意：`cin >>` 会跳过空格和换行，只读一个词。如果要读一整行（包括空格），用 `getline(cin, str)`。**

## 【实战1-2】用变量描述玩家信息

**任务：** 定义玩家名字、等级、HP、是否在线，打印出来

```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    // 玩家数据
    string player_name = "战士";
    int player_level = 30;
    double player_hp = 100.0;
    bool is_online = true;

    // 打印
    cout << "===== 玩家信息 =====" << endl;
    cout << "名字: " << player_name << endl;
    cout << "等级: " << player_level << endl;
    cout << "HP: " << player_hp << endl;
    cout << "在线: " << (is_online ? "是" : "否") << endl;

    return 0;
}
```

**输出：**
```
===== 玩家信息 =====
名字: 战士
等级: 30
HP: 100
在线: 是
```

**注意：`is_online ? "是" : "否"` 是三元运算符，`?` 前为真取冒号前，为假取冒号后。**

## 【实战1-4】玩家升级

**任务：** 输入当前等级，打印升级后的等级

## 【实战1-5】计算玩家战斗力

**任务：** 输入攻击力、防御力、暴击率，计算综合战斗力

```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    int attack;
    int defense;
    double crit_rate;

    cout << "===== 战斗力计算 =====" << endl;
    cout << "攻击力: ";
    cin >> attack;

    cout << "防御力: ";
    cin >> defense;

    cout << "暴击率(0-1): ";
    cin >> crit_rate;

    // 战斗力 = 攻击 * 2 + 防御 + 暴击率 * 攻击
    double power = attack * 2 + defense + crit_rate * attack;

    cout << "\n综合战斗力: " << power << endl;

    return 0;
}
```

## 本章总结

| 概念 | 代码 | 含义 |
|------|------|------|
| 变量声明 | `类型 名字 = 值;` | 先声明类型，再使用 |
| 字符串 | `string name = "战士";` | 需要 `#include <string>` |
| cout 输出 | `cout << "Hello" << endl;` | 打印到屏幕，`endl` 换行 |
| cin 输入 | `cin >> name;` | 从键盘读取到变量 |
| 注释 | `// 单行` 或 `/* 多行 */` | 给自己看的说明 |
| 三元运算符 | `a ? b : c` | a为真取b，为假取c |
