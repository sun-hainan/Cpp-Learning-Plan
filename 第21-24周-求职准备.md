## 三、面试高频题

### 题目1：new 和 delete 的原理

**标准回答：**
- `new` 先调用 `operator new` 分配内存，再调用构造函数
- `delete` 先调用析构函数，再调用 `operator delete` 释放内存
- `new[]` 和 `delete[]` 配对用于数组

```cpp
int* p = new int(10);
// 等价于：
void* temp = operator new(sizeof(int));  // 分配内存
int* p = static_cast<int*>(temp);         // 转换类型
*p = 10;                                   // 构造
```

**追问：内存泄漏怎么办？**
- 用智能指针 `unique_ptr`、`shared_ptr`（C++11）
- 用 valgrind 或 Visual Studio 的内存检测工具

### 题目2：指针和引用的区别

**标准回答：**

| 对比 | 指针 | 引用 |
|------|------|------|
| 空值 | 可以为 nullptr | 必须绑定对象 |
| 可改变 | 可以指向别的地址 | 绑定后不能改变 |
| 语法 | 用 `*` 解引用 | 直接当变量用 |
| 维度 | 一级指针、多级指针 | 只有一级引用 |

```cpp
int a = 10;
int* p = &a;   // 指针，可以改变
int& r = a;    // 引用，不能改变绑定对象
```

**追问：什么时候用指针？**
- 需要表示"没有对象"时 → 用指针 nullptr
- 需要改变所指向的对象时 → 用指针
- 只是传递引用不想复制时 → 用引用

### 题目3：vector 扩容机制

**标准回答：**
- vector 底层是动态数组，大小不够时会自动扩容
- 扩容策略：通常每次扩大到 1.5 倍或 2 倍
- 扩容时重新分配内存，把所有元素拷贝过去，效率低

```cpp
vector<int> v;
v.push_back(1);  // 假设容量为1
v.push_back(2);  // 扩容到2，拷贝
v.push_back(3);  // 扩容到4，拷贝
v.push_back(4);  // 容量足够
```

**追问：怎么避免频繁扩容？**
- 预先分配：`v.reserve(100);` 一次性分配足够空间
- `shrink_to_fit()` 释放多余容量

### 题目4：多态的实现原理

**标准回答：**
- 每个有虚函数的类有一个**虚函数表（vtable）**
- 每个对象有一个**虚指针（vptr）**指向类的 vtable
- 调用虚函数时，通过 vptr 找到 vtable，再调用正确的函数

```cpp
class Base {
public:
    virtual void foo();  // 有虚函数，就有 vptr
};

class Derived : public Base {
public:
    void foo() override;  // 重写，用新的函数地址替换 vtable 中的条目
};
```

**追问：构造函数能是虚函数吗？**
- 不能。构造时对象还没创建完，没有 vptr。

### 题目5：static 关键字的作用

| 用法 | 作用 |
|------|------|
| static 局部变量 | 函数结束后不销毁，下次调用保留值 |
| static 全局变量 | 只在当前文件可见 |
| static 成员变量 | 类的所有对象共享，只有这一个副本 |
| static 成员函数 | 属于类不属于对象，可以不用创建对象就调用 |

```cpp
class Player {
private:
    static int playerCount;  // 所有玩家共享的计数器

public:
    static int getCount() { return playerCount; }
};
```

### 题目6：智能指针

**三种智能指针：**

| 类型 | 特点 | 适用场景 |
|------|------|---------|
| `unique_ptr` | 独占所有权，不能复制 | 独占资源，如文件句柄 |
| `shared_ptr` | 引用计数，复制时计数+1 | 共享资源 |
| `weak_ptr` | 弱引用，不增加计数 | 打破循环引用 |

```cpp
#include <memory>

// 独占指针（推荐优先使用）
unique_ptr<int> p1 = make_unique<int>(10);
unique_ptr<int> p2 = p1;  // 错误！不能复制

// 共享指针
shared_ptr<int> p3 = make_shared<int>(10);
shared_ptr<int> p4 = p3;  // 正确，引用计数=2
```

**追问：和 Lua 的 GC 有什么区别？**
- 智能指针：引用计数，谁最后用完谁释放
- Lua GC：标记-清除，可能延迟释放，但不会遗漏

### 题目7：进程和线程的区别

**标准回答：**

| 对比 | 进程 | 线程 |
|------|------|------|
| 独立资源 | 有独立的地址空间 | 共享进程的地址空间 |
| 开销 | 大（创建要分配资源） | 小（共享资源） |
| 通信 | 靠 IPC（管道、消息队列等） | 直接读写共享内存 |
| 稳定性 | 一个进程崩溃不影响其他 | 一个线程崩溃整个进程崩溃 |

**游戏服务器通常用多线程：**
```
主线程：监听端口，接受连接
IO线程：处理网络读写
游戏逻辑线程：执行业务逻辑
```
